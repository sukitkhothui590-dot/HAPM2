ส่วนที่ 1 คู่มือสำหรับผู้ใช้งานทั่วไป (Public User Guide)

คู่มือฉบับนี้จัดทำขึ้นเพื่ออธิบายขั้นตอนการใช้งานเว็บไซต์ happympm.com สำหรับผู้เข้าชมทั่วไปและสมาชิก โดยครอบคลุมฟีเจอร์หลักทั้งหมดของระบบ

1.1 การเข้าถึงเว็บไซต์และการนำทาง (Navigation)

  • เปิด Browser พิมพ์ happympm.com เพื่อเข้าสู่หน้าหลัก
  • เมนูนำทางด้านบนรองรับการเข้าถึงทุกส่วนของเว็บไซต์
  • คลิกปุ่มเลือกภาษา (มุมบนขวา) เพื่อเปลี่ยนเป็น ไทย / English / Indonesia / Melayu / عربي
  • ระบบรองรับ Responsive บนทุกอุปกรณ์ ทั้ง Desktop, Tablet และ Mobile
🖥️ ภาพที่ ก.1 — หน้าจอหลักของเว็บไซต์ (Homepage) แคป happympm.com เต็มหน้าจอ แสดง Hero Section (Banner/Video), เมนูด้านบน, ข้อมูลผลิตภัณฑ์ และ AI Chat Widget มุมขวาล่าง ขนาดแนะนำ: 1440×900px — แคปเต็มจอ Desktop

ภาพที่ ก.1 หน้าจอหลักของเว็บไซต์ happympm.com

1.2 การใช้งานแผนที่สาขาอัจฉริยะ (Interactive Branch Map)

  • คลิกเมนู "แผนที่สาขา" เพื่อเข้าสู่ระบบแผนที่
  • คลิกจังหวัดบนแผนที่เพื่อดูรายละเอียดสาขา ที่อยู่ และเบอร์โทรศัพท์
  • สลับมุมมองระหว่าง แผนที่ประเทศไทย และ Globe View ด้วยปุ่มด้านบน
  • รองรับข้อมูลสาขาในไทย ลาว และกัมพูชา
🗺️ ภาพที่ ก.2 — ระบบแผนที่สาขาแบบ Interactive แคปหน้า /branch-map แสดงแผนที่ประเทศไทย (amCharts 5) พร้อมหมุดสาขา และ Pop-up ข้อมูลสาขาเมื่อคลิก ขนาดแนะนำ: 1280×720px

ภาพที่ ก.2 ระบบแผนที่สาขาแบบ Interactive แสดงสาขาทั่วประเทศ

1.3 การสนทนากับ AI Chat Assistant

  • คลิกไอคอนแชท (มุมล่างขวา) เพื่อเริ่มต้นการสนทนา
  • กรอกข้อมูล ชื่อ, เบอร์โทรศัพท์, อีเมล
  • เลือกหมวดหมู่ที่ต้องการสอบถาม: (1) สมัครธุรกิจ (2) ฝ่ายสนับสนุน (3) ผลิตภัณฑ์ (4) เรื่องอื่นๆ
  • พิมพ์คำถามแล้วกด Enter หรือปุ่มส่ง AI จะตอบกลับภายใน 5 วินาที
🤖 ภาพที่ ก.3 — AI Chat Widget และหน้าจอการสนทนา แคป 2 ภาพ — ซ้าย: ปุ่ม Chat ที่มุมล่างขวา / ขวา: กล่อง Chat เปิดขึ้นมาพร้อมข้อความสนทนา 2-3 รอบ ขนาดแนะนำ: 400×600px (เฉพาะ Chat Box)

ภาพที่ ก.3 AI Chat Widget และหน้าจอการสนทนากับระบบ AI

ส่วนที่ 2 คู่มือสำหรับผู้ดูแลระบบ (Admin CMS Guide)

2.1 การเข้าสู่ระบบ (Login)

  • เปิด Browser ไปที่ happympm.com/ADMIN-CMS
  • กรอก Email และ Password ที่ได้รับสิทธิ์ จากนั้นคลิก "เข้าสู่ระบบ"
  • ระบบจะ Redirect ไปยัง Dashboard อัตโนมัติตามระดับสิทธิ์ที่กำหนด
🔐 ภาพที่ ก.4 — หน้าจอเข้าสู่ระบบ Admin (Login Form) แคปหน้า Login ที่ happympm.com/ADMIN-CMS แสดง Form Email/Password, ปุ่ม Login และโลโก้บริษัท ขนาดแนะนำ: 900×600px

ภาพที่ ก.4 หน้าจอเข้าสู่ระบบสำหรับผู้ดูแลระบบ

2.2 Dashboard หลัก

  • หลัง Login จะเข้าสู่หน้า Dashboard แสดงสถิติสรุป
  • แสดงจำนวนผู้เข้าชมวันนี้, สัปดาห์นี้, เดือนนี้
  • แสดงจำนวน AI Chat Session ทั้งหมดและล่าสุด
  • เมนูด้านซ้ายรายการเมนูทั้งหมดตามระดับสิทธิ์ที่มี
📊 ภาพที่ ก.5 — หน้า Dashboard หลักของ Admin CMS แคปหน้า Dashboard แสดง Sidebar เมนูซ้าย, การ์ดสถิติ 4 ใบ และกราฟภาพรวม ขนาดแนะนำ: 1440×900px — แคปเต็มหน้าจอ

ภาพที่ ก.5 หน้า Dashboard หลักแสดงสถิติภาพรวมระบบ

2.3 การแก้ไขเนื้อหาแบบ Real-time (Live Editor)

  • คลิก "Live Editor" ในเมนูซ้าย
  • เลือกหน้าที่ต้องการแก้ไขจาก Dropdown ด้านบน
  • Panel ซ้าย: รายการ Section ที่แก้ไขได้ (ข้อความ, รูปภาพ, ลิงก์)
  • Panel ขวา: แสดง Preview เว็บไซต์จริงแบบ Real-time ผ่าน iframe
  • คลิก "บันทึก" เพื่อบันทึกการเปลี่ยนแปลง ผลจะแสดงบนเว็บทันที
✏️ ภาพที่ ก.6 — Live Editor พร้อม Preview Real-time แคปเต็มหน้าจอขณะใช้ Live Editor แสดง Panel แก้ไขด้านซ้าย และ iframe Preview เว็บไซต์จริงด้านขวา ขนาดแนะนำ: 1440×900px

ภาพที่ ก.6 หน้า Live Editor แสดงการแก้ไขเนื้อหาแบบ Real-time

2.4 การจัดเรียง Section (Section Order — Drag & Drop)

  • คลิก "จัดเรียง Section" ในเมนูซ้าย
  • เลือกหน้าที่ต้องการจัดลำดับ (Home, About ฯลฯ)
  • ลากวาง (Drag & Drop) แถบ Section เพื่อเปลี่ยนลำดับการแสดงผล
  • คลิก "บันทึกลำดับ" เพื่อยืนยัน ระบบจะอัปเดตหน้าเว็บทันที
↕️ ภาพที่ ก.7 — ระบบ Section Order (Drag & Drop) แคปหน้า Section Order แสดงรายการ Section ที่ลากได้ (มี Handle icon ด้านซ้าย) และหมายเลขลำดับ ขนาดแนะนำ: 900×700px

ภาพที่ ก.7 ระบบจัดลำดับ Section แบบ Drag & Drop

2.5 การจัดการฐานความรู้และตั้งค่า AI (AI Knowledge Base & Settings)

  • คลิก "จัดการ AI" หรือ "ฐานความรู้" ในเมนูซ้าย
  • เพิ่ม/แก้ไข/ลบบทความความรู้แต่ละหมวดหมู่ (สมัครธุรกิจ, ผลิตภัณฑ์, สนับสนุน, ทั่วไป)
  • หน้า "ตั้งค่า AI": เลือกโมเดล (GPT-4o / GPT-4o-mini / GPT-4.1-mini / GPT-4.1-nano)
  • ปรับค่า Temperature (0.0 = ตอบตามความรู้ตรงๆ / 1.0 = สร้างสรรค์มากขึ้น)
  • แก้ไข System Prompt เพื่อกำหนดบุคลิกและขอบเขตการตอบ
🧠 ภาพที่ ก.8 — หน้าจัดการฐานความรู้ AI และหน้าตั้งค่า แคป 2 ภาพ — (1) หน้ารายการ Knowledge Base พร้อมปุ่มเพิ่ม/แก้ไข/ลบ (2) หน้า AI Settings แสดง Dropdown โมเดลและ Slider Temperature ขนาดแนะนำ: 1280×720px

ภาพที่ ก.8 หน้าจัดการฐานความรู้ AI และการตั้งค่าโมเดล

2.6 Analytics Dashboard — สถิติผู้เข้าชม 77 จังหวัด

  • คลิก "Analytics" หรือ "สถิติผู้เข้าชม" ในเมนูซ้าย
  • ดูสถิติแบบ Real-time: จำนวน Page Views, Unique Visitors รายวัน/สัปดาห์/เดือน
  • กราฟจังหวัด: แสดง Top 20 จังหวัดที่เข้าชมมากที่สุด
  • แผนที่ Heatmap: แสดงความเข้มข้นของผู้ชมในแต่ละจังหวัด
📈 ภาพที่ ก.9 — Analytics Dashboard แผนที่ 77 จังหวัด แคปหน้า Analytics Dashboard เต็มจอ แสดงกราฟเส้นแนวโน้มผู้เข้าชม, กราฟแท่งจังหวัด Top 20 และตาราง ขนาดแนะนำ: 1440×900px

ภาพที่ ก.9 Analytics Dashboard แสดงสถิติผู้เข้าชมจำแนกตามจังหวัด

2.7 ระบบจัดการไฟล์มีเดีย (Media Library)

  • คลิก "จัดการมีเดีย" ในเมนูซ้าย
  • คลิก "Upload" เพื่ออัปโหลดรูปภาพ (รองรับ jpg, png, webp, gif ขนาดไม่เกิน 10MB)
  • คลิกรูปภาพเพื่อ Copy URL นำไปใช้ใน Live Editor
  • คลิก "ลบ" เพื่อลบไฟล์ที่ไม่ใช้งาน
🖼️ ภาพที่ ก.10 — Media Library แสดงรายการรูปภาพทั้งหมด แคปหน้า Media Library แสดง Grid รูปภาพ พร้อมปุ่ม Upload ด้านบน และ Hover Effect แสดง URL ขนาดแนะนำ: 1280×720px

ภาพที่ ก.10 หน้า Media Library สำหรับจัดการไฟล์รูปภาพ

2.8 ระบบจัดการคำแปล (Translation Manager)

  • คลิก "จัดการคำแปล" ในเมนูซ้าย
  • ระบบแสดงรายการคำแปลแต่ละ Key แยกตาม Namespace (เมนู, ปุ่ม, ข้อความ)
  • คลิกช่อง TH / EN / ID / MS / AR เพื่อแก้ไขคำแปลแต่ละภาษา
  • คลิก "บันทึก" เพื่อยืนยัน คำแปลจะอัปเดตบนเว็บไซต์ทันที
🌐 ภาพที่ ก.11 — Translation Manager แสดงตารางคำแปล 5 ภาษา แคปหน้า Translation Manager แสดงตารางมีคอลัมน์ Key, TH, EN, ID, MS, AR พร้อมช่องแก้ไข ขนาดแนะนำ: 1440×800px

ภาพที่ ก.11 หน้า Translation Manager สำหรับจัดการคำแปล 5 ภาษา

2.9 ระบบสำรองข้อมูล (Backup & Restore)

  • คลิก "Backup & Restore" ในเมนูซ้าย (ต้องมีสิทธิ์ระดับ Super Admin ขึ้นไป)
  • คลิก "Export ข้อมูล" เพื่อดาวน์โหลดข้อมูลทั้งหมดเป็นไฟล์ JSON
  • คลิก "Import / Restore" แล้วเลือกไฟล์ JSON เพื่อกู้คืนข้อมูล
  • คำเตือน: การ Restore จะเขียนทับข้อมูลปัจจุบัน กรุณาทำ Backup ก่อนเสมอ
💾 ภาพที่ ก.12 — หน้า Backup & Restore แคปหน้า Backup แสดงปุ่ม Export JSON, ปุ่ม Import/Restore และตารางประวัติการ Backup ที่ผ่านมา ขนาดแนะนำ: 1000×600px

ภาพที่ ก.12 หน้าระบบสำรองและกู้คืนข้อมูล

2.10 ระบบจัดการผู้ใช้งาน (User Management)

  • คลิก "จัดการผู้ใช้" ในเมนูซ้าย (ต้องมีสิทธิ์ระดับ Super Admin)
  • เห็นรายชื่อผู้ใช้ทั้งหมด พร้อมอีเมล, ชื่อ, ระดับสิทธิ์ และวันที่สร้างบัญชี
  • คลิก "เพิ่มผู้ใช้" เพื่อสร้างบัญชีใหม่ กำหนด Role (Viewer/Editor/Admin/Super Admin)
  • คลิก "แก้ไข" เพื่อเปลี่ยน Role หรือข้อมูล / คลิก "ลบ" เพื่อลบบัญชี
👥 ภาพที่ ก.13 — หน้าจัดการผู้ใช้งาน (User Management) แคปหน้า User Management แสดงตารางผู้ใช้ทั้งหมด พร้อม Badge สีของแต่ละ Role และปุ่มแก้ไข/ลบ ขนาดแนะนำ: 1280×720px

ภาพที่ ก.13 หน้าจัดการผู้ใช้งานและระดับสิทธิ์

ส่วนที่ 4 ขั้นตอนการ Build และ Deploy ระบบ

4.1 การ Build สำหรับ Public Website

ขั้นตอนที่ 1 ตั้งค่า Environment Variables ใน .env.production.local
REACT_APP_MODE=public REACT_APP_SUPABASE_URL=https://[project-id].supabase.co REACT_APP_SUPABASE_ANON_KEY=[your-anon-key] REACT_APP_API_URL=https://[vercel-app].vercel.app
ขั้นตอนที่ 2 รัน Build Command
set NODE_OPTIONS=--max-old-space-size=4096 set REACT_APP_MODE=public npx react-scripts build
ขั้นตอนที่ 3 Upload ไฟล์ใน /build ไปยัง cPanel public_html/ พร้อม .htaccess

4.2 การ Build สำหรับ Admin CMS

ขั้นตอนที่ 1 เปลี่ยน REACT_APP_MODE เป็น admin
set NODE_OPTIONS=--max-old-space-size=4096 set REACT_APP_MODE=admin npx react-scripts build
ขั้นตอนที่ 2 Upload ไฟล์ใน /build ไปยัง cPanel public_html/ADMIN-CMS/ พร้อม .htaccess ที่มี RewriteBase /ADMIN-CMS/

4.3 การ Deploy API Server บน Vercel

ขั้นตอนที่ 1 Push โฟลเดอร์ /api ขึ้น GitHub Repository
ขั้นตอนที่ 2 สร้าง Project ใหม่บน Vercel เชื่อมกับ Repository
ขั้นตอนที่ 3 ตั้งค่า Environment Variables ใน Vercel Dashboard (OPENAI_API_KEY, SUPABASE_SERVICE_ROLE_KEY, ALLOWED_ORIGINS)
ขั้นตอนที่ 4 Deploy และทดสอบ /api/health endpoint
